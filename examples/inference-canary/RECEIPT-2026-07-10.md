## Evaluator correction

An earlier repeat labeled Cursor and OpenAI as drifted on the review case. Both outputs found the authorization bug and proposed the right fix. They described it as `IDOR` and access to accounts the user did not belong to, while the evaluator only recognized terms such as `authorization` and `cross-tenant`.

The predicate was too narrow. I added the missing equivalent language, wrote a regression test from the two outputs, and reset the baseline because the suite fingerprint changed.

These runs were minutes apart. They verify the comparison and receipt path; they do not claim week-over-week model drift.

## Reproduce

```sh
uv run arena canary \
  --config examples/inference-canary/config.json \
  --backends codex,cursor,anthropic,openai,google,openrouter,fireworks \
  --runs-dir out/inference-canary
```

Run the same command again. `arena canary` selects the latest compatible baseline and reports the changed lane and check IDs.
